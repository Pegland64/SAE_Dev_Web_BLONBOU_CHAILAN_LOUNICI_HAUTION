6. Script SQL pour créer les tables et insérer des enregistrements

CREATE TABLE Soiree (
    id_soiree INT PRIMARY KEY,
    date DATE NOT NULL,
    lieu VARCHAR(100) NOT NULL
);

CREATE TABLE Spectacle (
    id_spectacle INT PRIMARY KEY,
    nom_spectacle VARCHAR(100) NOT NULL,
    style VARCHAR(50) NOT NULL
);

CREATE TABLE Programme (
    id_soiree INT,
    id_spectacle INT,
    PRIMARY KEY (id_soiree, id_spectacle),
    FOREIGN KEY (id_soiree) REFERENCES Soiree(id_soiree),
    FOREIGN KEY (id_spectacle) REFERENCES Spectacle(id_spectacle)
);

Explication du script
Tables créées : Les tables Soiree, Spectacle et Programme représentent les entités principales du modèle.
Données insérées : Des enregistrements exemples sont fournis pour illustrer la structure des tables. Les tables Soiree et Spectacle contiennent des informations sur les soirées et les spectacles, et Programme relie les deux pour définir les spectacles de chaque soirée.
Ce modèle est optimisé pour les requêtes nécessaires, notamment pour filtrer les spectacles par date, lieu, ou style de musique.
